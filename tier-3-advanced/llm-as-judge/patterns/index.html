Patterns: LLM-as-Judge

Pattern 1: Rubric Judge — Score on Multiple Dimensions​

Ask the judge to score a response on a fixed rubric. Request structured JSON output so scores can be parsed programmatically.

import json
import anthropic

client = anthropic.Anthropic()
MODEL = "claude-3-haiku-20240307"

DEFAULT_RUBRIC = {
    "helpfulness": "How helpful is the response? (1=not helpful, 5=extremely helpful)",
    "accuracy": "How factually accurate? (1=many errors, 5=fully accurate)",
    "clarity": "How clear and well-organized? (1=confusing, 5=very clear)",
}

SCORE_PROMPT = """You are an expert evaluator. Score the following response on each rubric dimension.
Return ONLY a JSON object with integer scores 1-5. No explanation, no markdown.

Rubric:
{rubric}

Prompt: {prompt}
Response: {response}

JSON scores:"""

def score_response(prompt: str, response: str, rubric: dict = None) -> dict:
    """Score a response on each rubric dimension. Returns {dimension: int}."""
    if rubric is None:
        rubric = DEFAULT_RUBRIC

    rubric_text = "\n".join(f"- {k}: {v}" for k, v in rubric.items())
    formatted = SCORE_PROMPT.format(
        rubric=rubric_text,
        prompt=prompt,
        response=response,
    )
    api_response = client.messages.create(
        model=MODEL,
        max_tokens=256,
        temperature=0,
        messages=[{"role": "user", "content": formatted}]
    )
    raw = api_response.content[0].text.strip()
    return json.loads(raw)

# Usage
scores = score_response(
    prompt="Explain what gradient descent is.",
    response="Gradient descent adjusts model weights to minimize loss. It uses the gradient to find the downhill direction."
)
print(scores)  # {"helpfulness": 4, "accuracy": 5, "clarity": 4}

When to use: Single-response quality assessment. Run on every response in your eval dataset to get per-dimension scores you can track over time.

Pattern 2: Pairwise Judge with Position Debiasing​

Run the comparison twice — swap response order between runs — to cancel out position bias.

PAIRWISE_PROMPT = """You are an expert evaluator comparing two responses to the same prompt.
Decide which response is better overall.
Return ONLY one of: "A", "B", or "tie". No explanation.

Prompt: {prompt}

Response A:
{response_a}

Response B:
{response_b}

Your verdict (A / B / tie):"""

def pairwise_judge(prompt: str, response_a: str, response_b: str) -> str:
    """
    Compare two responses. Returns "A", "B", or "tie".
    Runs the judge twice (swapping order) to mitigate position bias.
    """
    def _judge(p: str, ra: str, rb: str) -> str:
        resp = client.messages.create(
            model=MODEL,
            max_tokens=10,
            temperature=0,
            messages=[{"role": "user", "content": PAIRWISE_PROMPT.format(
                prompt=p, response_a=ra, response_b=rb
            )}]
        )
        verdict = resp.content[0].text.strip().upper()
        return verdict if verdict in ("A", "B", "TIE") else "tie"

    # Run 1: A first
    verdict_1 = _judge(prompt, response_a, response_b)

    # Run 2: B first — swap labels
    verdict_2_raw = _judge(prompt, response_b, response_a)
    # Invert labels: if judge said "A" when B was shown first, that means B won
    if verdict_2_raw == "A":
        verdict_2 = "B"
    elif verdict_2_raw == "B":
        verdict_2 = "A"
    else:
        verdict_2 = "tie"

    # Aggregate: both agree → that winner; disagree → tie
    if verdict_1 == verdict_2:
        return verdict_1
    return "tie"

# Usage
winner = pairwise_judge(
    prompt="What is the capital of France?",
    response_a="Paris is the capital of France.",
    response_b="France has a capital city.",
)
print(winner)  # "A"

When to use: A/B testing two model versions, comparing two prompt variants, or ranking candidate responses in a pipeline. Always use the debiased version (two runs) in production.

Pattern 3: Calibration Against Human Labels​

Before deploying a judge at scale, measure how well it agrees with human labels on a held-out calibration set.

def calibrate_judge(
    calibration_set: list[dict],
    judge_fn,
) -> dict:
    """
    Measure judge agreement with human labels.
    Each item: {"prompt": str, "response": str, "human_score": int}
    Returns: {"agreement_rate": float, "mean_judge_score": float, "mean_human_score": float}
    """
    judge_scores = []
    human_scores = []

    for item in calibration_set:
        scores = judge_fn(item["prompt"], item["response"])
        # Use mean of all rubric dimensions as a single score
        judge_mean = sum(scores.values()) / len(scores)
        judge_scores.append(judge_mean)
        human_scores.append(item["human_score"])

    # Agreement rate: judge and human within 1 point on a 5-point scale
    agreements = sum(
        1 for j, h in zip(judge_scores, human_scores) if abs(j - h) <= 1
    )
    agreement_rate = agreements / len(calibration_set) if calibration_set else 0.0

    return {
        "agreement_rate": agreement_rate,
        "mean_judge_score": sum(judge_scores) / len(judge_scores),
        "mean_human_score": sum(human_scores) / len(human_scores),
        "n": len(calibration_set),
    }

# Usage
calibration_data = [
    {"prompt": "Explain DNA", "response": "DNA carries genetic information.", "human_score": 3},
    {"prompt": "What is Python?", "response": "Python is a high-level programming language.", "human_score": 4},
]
report = calibrate_judge(calibration_data, score_response)
print(f"Agreement rate: {report['agreement_rate']:.0%}")

When to use: Before relying on any LLM judge at scale. If agreement is below 60%, revisit the rubric, the judge model, or both.

Anti-Patterns​

Anti-patternWhat Goes WrongFix
Same model as generator and judgeSelf-enhancement bias: the model prefers its own phrasing and style, inflating its own scoresUse a separate, ideally stronger, model as judge (e.g., Claude Opus judging Haiku outputs)
Single pairwise run (no position debiasing)Position bias contaminates ~20-30% of results; A wins more often just because it appears firstAlways run pairwise twice with swapped order; call disagreements a tie
Judge without rubricScores are inconsistent across different runs; "quality" means something different each timeWrite an explicit rubric with clear anchors for each score level
No calibrationJudge may be systematically biased (e.g., always giving 4s); you rely on numbers you can't trustCalibrate on ≥50 human-labeled examples before deploying
Counting ties as disagreementsTies are actually a valid outcome (no clear winner); treating them as failures inflates noiseTreat "tie" as a first-class result, not a failure to judge
