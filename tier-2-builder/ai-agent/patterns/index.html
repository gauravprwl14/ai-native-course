Patterns: Full Agent Architecture

Pattern 1: Tool Registry​

Separate tool definitions (what the model sees) from tool functions (what actually runs). Keep them in sync.

from dataclasses import dataclass
from typing import Callable

# Tool functions — real implementations
def _web_search(query: str) -> str:
    # In production: call a search API
    return f"[Mock] Search results for: {query}"

def _calculator(expression: str) -> str:
    import math
    try:
        result = eval(expression, {"__builtins__": {}}, {"math": math})
        return str(result)
    except Exception as e:
        return f"Error: {e}"

def _read_file(path: str) -> str:
    try:
        return open(path).read()[:2000]  # limit to 2000 chars
    except FileNotFoundError:
        return f"Error: file not found: {path}"

# Tool registry — maps name → function
TOOL_FUNCTIONS: dict[str, Callable] = {
    "web_search": _web_search,
    "calculator": _calculator,
    "read_file": _read_file,
}

# Tool definitions — Anthropic JSON schemas
TOOL_DEFINITIONS = [
    {
        "name": "web_search",
        "description": "Search the web for current information.",
        "input_schema": {
            "type": "object",
            "properties": {
                "query": {"type": "string", "description": "Search query"}
            },
            "required": ["query"]
        }
    },
    {
        "name": "calculator",
        "description": "Evaluate a mathematical expression.",
        "input_schema": {
            "type": "object",
            "properties": {
                "expression": {"type": "string", "description": "Python math expression"}
            },
            "required": ["expression"]
        }
    },
    {
        "name": "read_file",
        "description": "Read the contents of a local file.",
        "input_schema": {
            "type": "object",
            "properties": {
                "path": {"type": "string", "description": "Absolute or relative file path"}
            },
            "required": ["path"]
        }
    },
]

Pattern 2: Agent System Prompt Design​

Structure the system prompt with four sections: role, capabilities, constraints, output format.

AGENT_SYSTEM_PROMPT = """You are a research and analysis assistant.

## Capabilities
You have access to the following tools:
- web_search: Search the web for current information
- calculator: Evaluate mathematical expressions
- read_file: Read the contents of a local file
- take_note: Save a note to your scratchpad for later reference

## Constraints
- Do not fabricate facts. If you cannot find information with the available tools, say so.
- Do not read files outside of the /tmp/workspace directory.
- For any calculation that involves money or legal decisions, show your work step by step.
- If a task is ambiguous, ask one clarifying question before proceeding.

## Behavior
- Use web_search for facts that may be outdated in your training data (e.g., current prices, recent events).
- Use take_note to record key findings as you work — you can reference them later.
- Break complex tasks into smaller steps. Complete each step before moving to the next.

## Output Format
When you have completed your research, provide:
1. **Summary** (2-3 sentences)
2. **Key Findings** (bullet list)
3. **Sources** (which tools/queries were used)
"""

Pattern 3: Error Budget​

Track consecutive failures. Abort if too many tool errors occur in a row.

def run_with_error_budget(
    task: str,
    tools: list[dict],
    tool_functions: dict,
    max_iterations: int = 10,
    max_consecutive_errors: int = 3,
) -> str:
    import anthropic
    client = anthropic.Anthropic()
    messages = [{"role": "user", "content": task}]
    consecutive_errors = 0

    for i in range(max_iterations):
        response = client.messages.create(
            model="claude-3-haiku-20240307",
            max_tokens=1024,
            system=AGENT_SYSTEM_PROMPT,
            tools=tools,
            messages=messages,
        )

        if response.stop_reason == "end_turn":
            for block in response.content:
                if hasattr(block, "text"):
                    return block.text
            return ""

        if response.stop_reason == "tool_use":
            tool_block = next(b for b in response.content if b.type == "tool_use")

            # Execute tool
            if tool_block.name in tool_functions:
                try:
                    result = str(tool_functions[tool_block.name](**tool_block.input))
                except Exception as e:
                    result = f"Error: {e}"
            else:
                result = f"Error: Unknown tool '{tool_block.name}'"

            # Update error budget
            if result.startswith("Error:"):
                consecutive_errors += 1
                if consecutive_errors >= max_consecutive_errors:
                    return (
                        f"Agent aborted after {max_consecutive_errors} consecutive tool errors. "
                        f"Last error: {result}"
                    )
            else:
                consecutive_errors = 0  # reset on any success

            messages.append({"role": "assistant", "content": response.content})
            messages.append({
                "role": "user",
                "content": [{
                    "type": "tool_result",
                    "tool_use_id": tool_block.id,
                    "content": result,
                }]
            })

    return "Agent reached max iterations without completing the task."

Pattern 4: Agent Result Dataclass​

Return a structured result so callers know if the agent succeeded, how many steps it took, and what it did.

from dataclasses import dataclass, field

@dataclass
class AgentResult:
    """Structured return type for a completed agent run."""
    success: bool
    answer: str
    steps_taken: int
    tool_calls: list[dict] = field(default_factory=list)
    error: str | None = None

    def __str__(self) -> str:
        status = "SUCCESS" if self.success else "FAILED"
        return f"[{status}] {self.answer} ({self.steps_taken} steps)"


# Usage
result = AgentResult(
    success=True,
    answer="The answer is 42.",
    steps_taken=3,
    tool_calls=[
        {"tool": "calculator", "input": {"expression": "6*7"}, "result": "42"}
    ]
)

print(result)
# [SUCCESS] The answer is 42. (3 steps)
print(result.tool_calls[0]["tool"])
# calculator

Anti-Patterns​

Anti-PatternWhat Goes WrongFix
Vague system promptAgent hallucinates capabilities, gives inconsistent answersBe explicit: list exactly what tools exist and what the agent should do when uncertain
No error budgetAgent spams a broken tool until max_iterations, wasting budgetAdd consecutive_errors counter, abort after N
Mismatched tool registryTool definition says query param, function expects search_query — silent failureKeep definitions and functions in sync; test each tool independently
Returning raw messagesCallers have to parse the messages list to find the answerReturn a structured AgentResult from run()
