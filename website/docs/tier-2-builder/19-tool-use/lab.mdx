# Lab 19 — Build a Weather + Calculator Agent

## What You're Building

A complete tool-use agent with three functions:

1. **`define_tools()`** — returns the tool definitions for `get_weather` and `calculate` as a list of dicts with the correct JSON schema structure
2. **`execute_tool(tool_name, tool_input)`** — routes a tool call to the right function and returns a string result; handles unknown tools and calculation errors gracefully
3. **`run_agent(user_message, max_iterations=10)`** — runs the full tool use loop: sends messages, handles `tool_use` responses, returns tool results to the model, stops on `end_turn`

## Tools to Implement

### `get_weather`
- Parameter: `location` (string)
- Returns: `"The weather in {location} is 72°F and sunny."` (mock — no real API needed)

### `calculate`
- Parameter: `expression` (string, e.g. `"2 ** 32"` or `"sqrt(144)"`)
- Evaluates the expression using `eval` with `math.__dict__` in scope and `__builtins__` disabled
- Returns the result as a string
- On any error, returns `"Error: invalid expression"`

## Acceptance Criteria

- [ ] `define_tools()` returns a list of exactly 2 dicts
- [ ] Each tool dict has `"name"`, `"description"`, and `"input_schema"` keys
- [ ] `execute_tool("get_weather", {"location": "London"})` returns a string containing "London"
- [ ] `execute_tool("calculate", {"expression": "2+2"})` returns `"4"`
- [ ] `execute_tool("calculate", {"expression": "1/0"})` returns an error string (does not crash)
- [ ] `execute_tool("unknown_tool", {})` returns an error string for unknown tools
- [ ] `run_agent(message)` returns a string
- [ ] `run_agent` stops when `stop_reason == "end_turn"` and returns the text response
- [ ] `run_agent` handles `tool_use` by executing the tool and returning the result in the next message
- [ ] All tests pass: `pytest tests/ -v`
