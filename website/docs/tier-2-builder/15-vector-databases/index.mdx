# Chapter 15 — Vector Databases

## What You'll Learn

- Why in-memory cosine similarity breaks at scale and what vector databases solve
- How Approximate Nearest Neighbor (ANN) search works and why it's fast
- How HNSW indexing enables millisecond similarity search
- Chroma basics — creating collections, adding documents, querying
- Metadata filtering — narrow the search space before vector comparison
- Persistence — keeping your data alive across restarts

## Prerequisites

- Chapter 14 (RAG Core) — you should understand embeddings, chunking, and retrieval-augmented generation before working with vector databases
- Python 3.10+, `chromadb` and `openai` packages installed

## How This Connects

Chapter 14 showed you RAG — but used simple in-memory search. That works for dozens or hundreds of documents. Vector databases are what you reach for when your document corpus grows to thousands or millions of chunks. Every production RAG system — customer support bots, internal knowledge bases, document search — runs on a vector database underneath. Hybrid search (Chapter 16) layers keyword search on top of the vector retrieval you learn here.
