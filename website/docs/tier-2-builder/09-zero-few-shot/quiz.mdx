      ],
      correct: 1,
      explanation: "Label imbalance in few-shot examples directly biases the model. If all 10 examples are 'normal', the model sees 'normal' as overwhelmingly likely. It will predict 'normal' even for clearly toxic inputs. Always balance examples across classes — or weight them by real class frequency."

      correct: 1,

The model 'learns' to handle formal email language and applies that style to casual posts — hurting accuracy. Examples must represent the actual inputs you'll encounter in production."
