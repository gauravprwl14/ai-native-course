This concentrates relevant context and removes noise, which directly improves answer quality without changing retrieval or the LLM itself."

      correct: 1,
      explanation: "Cross-encoders cannot pre-compute document representations at index time — the encoding depends on the query. Running a cross-encoder against every document in a million-document corpus at query time would take minutes. Bi-encoders pre-embed documents at index time, so query-time retrieval is just one embedding call plus ANN search — O(1) against the index size. The two-stage pipeline gets the best of both: fast retrieval plus precise re-scoring."

In this case, you should increase the initial retrieval size (e.g., top-100) or investigate whether the embedding model captures the query-document relationship well. The re-ranker is only as good as the candidates it receives."
