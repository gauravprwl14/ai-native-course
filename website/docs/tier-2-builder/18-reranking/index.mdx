# Chapter 18 — Re-ranking

## What You'll Learn

- Why vector search alone returns noisy candidates and what a re-ranker fixes

## How This Connects

Chapter 14 built a RAG pipeline that embeds a query, retrieves the top-k chunks by cosine similarity, and sends them to the LLM. The problem: cosine similarity is a rough proxy for relevance. The top chunk by embedding distance isn't always the most useful chunk for answering the question.

Re-ranking adds a second stage between retrieval and generation. You retrieve a large candidate set cheaply (bi-encoder), then re-score with a much more precise model (cross-encoder or LLM). Only the best-scoring chunks reach the LLM. This is the difference between a good RAG demo and a production-quality RAG system.
