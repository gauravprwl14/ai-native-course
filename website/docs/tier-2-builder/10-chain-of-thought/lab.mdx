# Lab 10 — Solve Math Word Problems with CoT

## What You're Building

A Python module that solves math word problems using chain-of-thought prompting. You will implement:

1. **`solve_with_cot`** — Zero-shot CoT: prompt the model to think step by step and return the full reasoning response
2. **`extract_answer`** — Parse the final numerical answer from a CoT response using regex
3. **`solve_with_self_consistency`** — Generate N reasoning chains and take a majority vote
4. **`evaluate_math_accuracy`** — Measure what fraction of predictions match the correct answers

## Acceptance Criteria

- [ ] `solve_with_cot(problem)` returns a string containing the model's full step-by-step reasoning
- [ ] `extract_answer(response)` correctly parses the number from "Therefore, the answer is: X"
- [ ] `extract_answer` returns `None` when the pattern is not found
- [ ] `solve_with_self_consistency(problem, n)` calls the API exactly `n` times
- [ ] `solve_with_self_consistency` returns the most frequently occurring extracted answer
- [ ] `evaluate_math_accuracy` returns 1.0 when all predictions match, 0.0 when none match
- [ ] `None` predictions (failed extractions) count as wrong in `evaluate_math_accuracy`
- [ ] All tests pass: `pytest tests/ -v`

## Key Concepts You'll Use

- `COT_PROMPT.format(problem=...)` — format the zero-shot CoT prompt template
- `client.messages.create(temperature=0)` — deterministic generation for single-chain CoT
- `client.messages.create(temperature=0.7)` — stochastic generation for self-consistency
- `re.search(pattern, text)` — extract the answer with a regex pattern
- `collections.Counter` — find the most common answer across N chains

## Extend the Lab (Optional)

Once your tests pass, try these extensions:
