      explanation: "A max_iterations counter is the standard safety mechanism. Without it, a confused agent (or a model caught in a reasoning rut) will loop forever, burning API budget. Set a hard limit (e.g., 10) and return a graceful failure message when it's hit."

      id: "q3",
      text: "Why must the Observation (tool result) be added back to the conversation before the next Think step?",

        "C) Tools are stateless so results are lost otherwise",
        "D) Observations are optional — the model can infer the result"
      ],
      correct: 1,
      explanation: "LLMs are stateless. Every call to the API starts fresh from the messages you provide. If you don't append the tool result to the conversation, the model has no idea what the tool returned — it'll either repeat the same action or hallucinate an answer. The messages list IS the agent's memory."

        "D) Increase max_tokens"

      correct: 1,
      explanation: "Wrap every tool call in try/except and return an error string as the observation (e.g., 'Error: connection timeout'). The model receives the error in context and can adapt — retry with different args, try a different tool, or tell the user it couldn't complete the task. This is the error recovery pattern."
