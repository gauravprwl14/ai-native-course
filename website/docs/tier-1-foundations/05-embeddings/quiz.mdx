      explanation: "Keyword search (like BM25 or basic string matching) compares characters. 'puppy' and 'dog' are completely different strings, so a keyword search for 'puppy' won't return a document that only mentions 'dog'. Embeddings encode meaning — both words end up near each other in vector space, so embedding search finds the connection."

Instead: embed all descriptions once, store the vectors, then at query time embed only the query (1 API call) and run cosine similarity against the cached vectors. This is O(1) API calls per query regardless of corpus size."
