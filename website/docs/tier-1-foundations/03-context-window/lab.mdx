# Lab 03 — Build a Context-Aware Q&A System

## What You're Building

A Python module that manages conversation history for a multi-turn chatbot. The system must stay within a configurable token budget, automatically truncating old messages when the context fills up.

## Function Signatures

### `count_history_tokens(history: list[dict]) -> int`

Count total tokens for a list of message dicts. Each dict has `role` and `content` keys.

```python
history = [
    {"role": "user", "content": "What is Python?"},
    {"role": "assistant", "content": "Python is a programming language."},
]
count_history_tokens(history)  # returns ~20
```

### `truncate_history(history: list[dict], max_tokens: int) -> list[dict]`

Remove the oldest messages until the history fits within `max_tokens`. Return a new list — do not mutate the original.

```python
truncate_history(history, max_tokens=10)
# Returns history with oldest messages removed to fit within 10 tokens
```

### `chat_with_history(user_message, history, max_tokens=4000) -> tuple[str, list[dict]]`

Send a message to Claude with managed conversation history.

```python
response, new_history = chat_with_history(
    "What is the capital of France?",
    [],
)
# response = "Paris is the capital of France."
# new_history = [
#   {"role": "user", "content": "What is the capital of France?"},
#   {"role": "assistant", "content": "Paris is the capital of France."},
# ]
```
