# Chapter 46 — AI Gateway: Unified LLM Proxy

## What You'll Learn

As organizations adopt multiple LLMs, directly calling each provider API from every service becomes unmanageable. An AI gateway is the central control plane that sits between your application and the LLM providers, giving you routing, policy enforcement, cost visibility, and operational resilience in one place.

## What You'll Learn

- **Why gateways exist** — the operational chaos of multi-provider LLM usage at scale
- **Routing** — how the gateway maps model names to the right provider and backend
- **Load balancing** — spreading requests across multiple endpoints for reliability
- **Cost tracking** — per-team, per-project, per-user attribution of LLM spend
- **Rate limiting** — enforcing per-key and per-team quotas at the gateway level
- **Audit logging** — capturing every request and response for compliance and debugging
- **Provider fallback** — automatic failover when a provider returns errors
- **Circuit breakers** — stopping traffic to unhealthy backends to prevent cascade failures
