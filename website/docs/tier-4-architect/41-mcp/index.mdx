## How This Connects

Chapter 19 showed you how a single LLM can call tools defined in the same codebase. That works well for a single application — but it means every team reinvents the same integration plumbing: "how does the model get access to my database? my file system? my internal APIs?"

MCP answers that question once, for everyone. It is an open standard — led by Anthropic but now multi-vendor — that separates the AI host (the thing running the model) from the data servers (the things that provide tools and data). Once you write an MCP server for your database, any MCP-compatible host can use it: Claude Desktop, Cursor, VS Code Copilot, or your own custom app.

This chapter builds the mental model and hands-on skills you need to write MCP servers and understand why they are becoming the default integration layer for production AI systems.
