# Lab 41 â€” Build an MCP Server

In this lab you will implement a simplified MCP server from scratch using only Python's standard library. The goal is to understand MCP mechanics without depending on the `mcp` package â€” the patterns you implement here are identical to what the real SDK does under the hood.

## What You'll Build

| Component | Description |
|-----------|-------------|
| `MCPTool` | Dataclass holding tool name, description, input schema, and handler function |
| `MCPResource` | Dataclass holding resource URI, name, description, and MIME type |
| `MCPServer` | Server class with `register_tool`, `register_resource`, `list_tools`, `list_resources`, and `call_tool` |
| `create_filesystem_server` | Factory function that creates a server with `list_files` and `read_file` tools |

## Your Tasks

### Task A: `MCPServer.register_tool`

Store the tool in `self._tools` keyed by `tool.name`.

### Task B: `MCPServer.register_resource`

Store the resource in `self._resources` keyed by `resource.uri`.

### Task C: `MCPServer.list_tools`

Return a list of dicts in MCP format:
```python
[
    {
        "name": tool.name,
        "description": tool.description,
        "inputSchema": tool.input_schema,
    }
    for tool in self._tools.values()
]
```

### Task D: `MCPServer.list_resources`

Return a list of dicts:
```python
[
    {
        "uri": r.uri,
        "name": r.name,
        "description": r.description,
        "mimeType": r.mime_type,
    }
    for r in self._resources.values()
]
```

### Task E: `MCPServer.call_tool`

1. Look up the tool by name; raise `ValueError` if not found
2. Call `tool.handler(arguments)` and return `{"content": result, "isError": False}`
3. Catch any exception and return `{"content": str(e), "isError": True}`

### Task F: `create_filesystem_server`

1. Create an `MCPServer("filesystem")`
2. Create a `list_files` tool:
   - Name: `"list_files"`, Description: `"List files in the base directory"`
   - Handler: returns `sorted(os.listdir(base_dir))`
3. Create a `read_file` tool:
   - Name: `"read_file"`, Description: `"Read a file's contents"`
   - Input schema: `{"type": "object", "properties": {"filename": {"type": "string"}}}`
   - Handler: reads `base_dir/filename`, raises `ValueError` if path is outside `base_dir`
4. Register both tools; return the server

**Security requirement for `read_file`:** Use `os.path.realpath` to resolve symlinks, then check that the resolved path starts with `os.path.realpath(base_dir)`. Raise `ValueError("Access denied")` if not.

## Expected Behaviour

```python
server = create_filesystem_server("/tmp/test_dir")

# List tools
tools = server.list_tools()
assert any(t["name"] == "list_files" for t in tools)
assert any(t["name"] == "read_file" for t in tools)

# Call list_files
result = server.call_tool("list_files", {})
assert result["isError"] is False
assert isinstance(result["content"], list)

# Security: path traversal blocked
result = server.call_tool("read_file", {"filename": "../../etc/passwd"})
assert result["isError"] is True
assert "Access denied" in result["content"]
```
