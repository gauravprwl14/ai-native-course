# Chapter 43 — LLM Hosting: Running Models Locally

## What You'll Learn

In this chapter you'll move beyond cloud API calls and learn how to run large language models on your own hardware. You'll understand the full spectrum of self-hosting options, how quantization shrinks models to fit on consumer hardware, and how Ollama gives you an OpenAI-compatible API on localhost in minutes.

## Prerequisites

- Chapters 39–42 complete
- Basic familiarity with REST APIs and Python HTTP clients

## What You'll Learn

- **Self-hosted vs API trade-offs** — when each option wins on cost, privacy, and latency
- **Ollama** — the easiest way to run open-weight models locally with a REST API
- **OpenAI-compatible APIs** — drop-in replacement so your existing code just works
- **Quantization** — how FP32 → FP16 → INT8 → INT4 GGUF shrinks model size vs quality
- **GGUF format** — the llama.cpp portable model format, CPU-friendly and widely supported
- **Memory requirements** — how much RAM/VRAM you actually need for each model size

## How This Connects

Self-hosting is one pillar of the production LLM architecture. It pairs with caching (Chapter 44) and latency optimization (Chapter 45) to give you full control over cost, privacy, and performance.
