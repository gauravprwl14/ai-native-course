## How This Connects

Chapter 24 showed you how to build a multi-agent system where agents are defined in the same codebase: an orchestrator hardcodes which specialists exist and how to call them. That works for a self-contained project — but it breaks the moment agents need to be:

- **Discovered dynamically**: "I need an agent that can translate Korean, who is available right now?"
- **Deployed independently**: each specialist agent runs in its own service, possibly on different machines
- **Composed across organisations**: your orchestrator agent uses a translation specialist built by a different team

A2A and ACP are the protocols that make this possible. They define how agents publish their capabilities, how other agents discover them, and how tasks get delegated and results returned.
