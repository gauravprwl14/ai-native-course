# Chapter 49 — Computer Use: UI Agents

## What You'll Learn

Many valuable tasks require a user interface that has no programmatic API. Filing a form on a legacy government website, interacting with a desktop application, navigating a UI that changes too fast to maintain a scraper — these can only be automated by an agent that sees and acts on the screen exactly like a human would.

Computer use is the capability that enables this: an LLM that receives screenshots and decides what to click, type, or scroll.

- **Sandboxing** — why running computer use agents without a sandbox is a security risk
- **Safety guardrails** — human-in-the-loop approval for destructive actions
- **UI agent challenges** — visual understanding, coordinate precision, dynamic UIs, infinite loops
