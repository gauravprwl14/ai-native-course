# Patterns: Hallucination Detection

## Pattern 1: LLM Faithfulness Checker

The simplest production-grade approach: ask an LLM judge to score the answer against the context. Returns a float 0–1 and a reason.

```python
import json
from utils import get_anthropic_client

FAITHFULNESS_PROMPT = """Given a context and an answer, rate how faithful the answer is to the context.
Return a JSON object: {{"score": float 0-1, "reason": "brief explanation"}}
A score of 1.0 means every claim in the answer is supported by the context.
A score of 0.0 means the answer directly contradicts or invents information not in the context.

Context: {context}
Answer: {answer}"""

def check_faithfulness(answer: str, context: str) -> float:
    client = get_anthropic_client()
    prompt = FAITHFULNESS_PROMPT.format(context=context, answer=answer)
    response = client.messages.create(
        model="claude-3-haiku-20240307",
        max_tokens=256,
        messages=[{"role": "user", "content": prompt}]
    )
    data = json.loads(response.content[0].text)
    return float(data["score"])

# Usage
score = check_faithfulness(
    answer="Refunds are accepted within 60 days.",
    context="Refunds are accepted within 30 days of purchase."
)
print(score)  # ~0.1 — contradiction detected
```

**When to use**: Any RAG pipeline as the primary quality gate. Fast (one API call), interpretable (reason field explains the score).

## Pattern 2: Sentence-Level Verification

Break the answer into individual sentences and check each one independently. Gives you a list of which specific claims are contradicted or unsupported.

```python
import json
from utils import get_anthropic_client

CONTRADICTION_PROMPT = """You are a fact-checker. Given a context and an answer, identify every sentence in the answer that:
1. Directly contradicts something stated in the context, OR
2. States a fact that is not mentioned anywhere in the context

Return a JSON list of suspicious sentences. Return an empty list [] if everything is supported.

Context: {context}
Answer: {answer}"""

def detect_contradictions(answer: str, context: str) -> list[str]:
    client = get_anthropic_client()
    prompt = CONTRADICTION_PROMPT.format(context=context, answer=answer)
    response = client.messages.create(
        model="claude-3-haiku-20240307",
        max_tokens=512,
        messages=[{"role": "user", "content": prompt}]
    )
    text = response.content[0].text.strip()
    # Parse the JSON list from the response
    start = text.find("[")
    end = text.rfind("]") + 1
    if start == -1:
        return []
    return json.loads(text[start:end])

# Usage
issues = detect_contradictions(
    answer="Refunds are accepted within 60 days. No receipt is needed.",
    context="Refunds are accepted within 30 days of purchase with a valid receipt."
)
print(issues)
# ["Refunds are accepted within 60 days.", "No receipt is needed."]
```

**When to use**: Debugging pipelines, producing user-facing citations, and when you need to know *which* claim is wrong, not just that something is wrong.

## Pattern 3: Self-Consistency Check

Generate the same answer multiple times at temperature > 0. Compare answers for agreement. High disagreement signals uncertainty and potential hallucination.

```python
def self_consistency_check(question: str, context: str, n: int = 3) -> dict:
    """Generate n answers and measure consistency."""
    client = get_anthropic_client()
    answers = []
    for _ in range(n):
        response = client.messages.create(
            model="claude-3-haiku-20240307",
            max_tokens=256,
            temperature=0.7,
            messages=[{
                "role": "user",
                "content": f"Context: {context}\n\nQuestion: {question}\n\nAnswer briefly."
            }]
        )
        answers.append(response.content[0].text.strip())

    # Check pairwise faithfulness between answers
    # High variance = uncertain = potential hallucination
    unique_answers = len(set(a.lower()[:100] for a in answers))
    consistency_score = 1.0 - (unique_answers - 1) / max(n - 1, 1)

    return {
        "answers": answers,
        "consistency_score": consistency_score,
        "likely_reliable": consistency_score >= 0.7
    }
```

**When to use**: High-stakes queries where you need confidence beyond a single generation. Expensive (N API calls) — use sparingly.

## Anti-Patterns

| Anti-Pattern | Why It's a Problem | Better Approach |
|---|---|---|
| **Running faithfulness check after returning the answer** | Users already saw the hallucinated content | Integrate check into the pipeline *before* returning |
| **Using a threshold of 0.5** | Half the claims can be wrong and still "pass" | Start at 0.8; tune against a labeled evaluation set |
| **Only checking the full answer, not individual sentences** | You know something is wrong but can't fix it | Use `detect_contradictions` alongside the global score |
| **Skipping the check when context is long** | Long context = more surface area for hallucination | Check is more important, not less, for long contexts |
| **Trusting self-consistency alone** | Model can be consistently wrong (systematic bias) | Use self-consistency as a supplement, not a replacement for faithfulness checking |
