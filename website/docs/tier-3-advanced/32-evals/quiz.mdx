      explanation: "ROUGE-L uses the Longest Common Subsequence (LCS) to measure overlap. Unlike BLEU (which measures n-gram precision) or edit distance (which counts character-level edits), LCS captures the longest sequence of tokens that appear in both prediction and reference in the same order, allowing for paraphrasing while rewarding structural similarity."

      ],
      correct: 1,
      explanation: "Regression testing compares the current eval score to a stored baseline. If a code change — such as updating the prompt, changing chunk size, or swapping the model — causes the score to drop by more than a defined threshold, the eval fails, flagging a regression. This is directly analogous to a failing unit test in software engineering."

      explanation: "LLM-as-judge evals can score on rubric dimensions like helpfulness, coherence, and reasoning quality — qualities that ROUGE-L and exact match cannot capture. A response can score high on ROUGE-L but be misleading or unhelpful. An LLM judge with a good rubric can evaluate these subjective dimensions at scale, making it flexible for open-ended generation tasks."
