## How This Connects

Chapter 20 gave you a loop that can call tools repeatedly. But what happens when the task itself is too complex to reason about in a single context? "Write a full market analysis report" is too large for one LLM call. You need a layer above the agent loop — something that breaks the goal apart before execution begins.

Planning is that layer. A planner LLM turns a goal into an ordered list of sub-tasks. An executor then works through them one by one (or in parallel). If a step fails, the planner can regenerate the remaining steps with the new context. This is the foundation of every sophisticated autonomous system.
