# Lab: LoRA Dataset & Config Preparation

## Goal

Build the configuration and dataset preparation pipeline for a LoRA fine-tuning run. Actual training requires a GPU — this lab focuses on the pure Python logic that happens before training begins.

## What You'll Build

| Function | Description |
|----------|-------------|
| `create_lora_config(rank, alpha, target_modules)` | Returns a `LoRAConfig` dataclass with the given parameters |
| `format_for_trainer(examples)` | Converts `{"instruction", "output"}` dicts to `{"text": "### Instruction:\n...\n### Response:\n..."}` format |
| `estimate_trainable_params(total_model_params, rank, num_lora_layers, layer_dim)` | Estimates trainable LoRA parameters vs total and returns percentage |

## Acceptance Criteria

- `create_lora_config(8, 16, ["q_proj", "v_proj"])` returns a `LoRAConfig` with `rank=8`, `alpha=16`
- `config.scaling` returns `alpha / rank` (e.g., 16/8 = 2.0)
- `format_for_trainer([{"instruction": "...", "output": "..."}])` returns a list with a `"text"` key
- The `"text"` value starts with `"### Instruction:\n"`
- `estimate_trainable_params(7_000_000_000, 8, 12)` returns a dict with `"trainable"`, `"total"`, `"percentage"` keys
- `percentage` is approximately `(2 * 8 * 4096 * 12) / 7_000_000_000 * 100`
- All 8 tests pass: `pytest tests/ -v`

## Hints

1. `LoRAConfig.scaling` is a `@property` that returns `self.alpha / self.rank`
2. `format_for_trainer` builds the text using an f-string with `### Instruction:\n` and `### Response:\n` markers
3. `estimate_trainable_params`: each LoRA layer adds two matrices — A (layer_dim × rank) and B (rank × layer_dim). Per layer: `2 × rank × layer_dim` params
4. The `percentage` is `(trainable / total) * 100`, rounded to 4 decimal places
