# Patterns: Agent Memory

## Pattern 1: In-Context Memory — The Messages List

The simplest pattern. Maintain the full conversation history in the `messages` list and pass it to the model on every call.

```python
from anthropic import Anthropic

client = Anthropic()
MODEL = "claude-3-haiku-20240307"

def chat_with_memory(messages: list[dict], user_input: str) -> str:
    """Add user message, call LLM, append assistant reply, return reply."""
    messages.append({"role": "user", "content": user_input})

    response = client.messages.create(
        model=MODEL,
        max_tokens=1024,
        messages=messages,
    )

    reply = response.content[0].text
    messages.append({"role": "assistant", "content": reply})
    return reply


# Usage — messages list is the memory
conversation = []
print(chat_with_memory(conversation, "My name is Alice."))
print(chat_with_memory(conversation, "What's my name?"))  # Remembers "Alice"
```

**When to use:** single-session conversations where history fits in the context window. Works out of the box — no extra infrastructure needed.

## Pattern 2: File-Based Persistent Memory

Write named facts to a JSON file. Load them at startup and inject the most relevant ones into the system prompt.

```python
import json
from pathlib import Path
from anthropic import Anthropic

client = Anthropic()
MODEL = "claude-3-haiku-20240307"
MEMORY_FILE = "agent_memory.json"


def load_memories(path: str = MEMORY_FILE) -> dict:
    """Load all memories from disk. Returns empty dict if file does not exist."""
    try:
        with open(path) as f:
            return json.load(f)
    except FileNotFoundError:
        return {}


def save_memory(key: str, value: str, path: str = MEMORY_FILE) -> None:
    """Persist a key-value memory to disk."""
    memories = load_memories(path)
    memories[key] = value
    with open(path, "w") as f:
        json.dump(memories, f, indent=2)


def build_system_prompt(memories: dict) -> str:
    """Inject all memories into the system prompt."""
    if not memories:
        return "You are a helpful assistant."
    facts = "\n".join(f"- {k}: {v}" for k, v in memories.items())
    return f"You are a helpful assistant. Here is what you know about the user:\n{facts}"


# Usage
save_memory("user_name", "Alice")
save_memory("preferred_language", "Python")
save_memory("current_project", "refactoring the billing service")

memories = load_memories()
system_prompt = build_system_prompt(memories)

response = client.messages.create(
    model=MODEL,
    max_tokens=512,
    system=system_prompt,
    messages=[{"role": "user", "content": "What am I working on?"}],
)
print(response.content[0].text)
# "You're working on refactoring the billing service."
```

**When to use:** small numbers of structured facts (< 50 memories). Simple, no external dependencies. Does not scale to large memory stores — inject-all is too noisy at scale.

## Pattern 3: Semantic Memory with Embedding Search

Embed memories as vectors. At query time, retrieve only the most relevant memories using cosine similarity. Scales to thousands of memories without polluting the context window.

```python
import math
import json
from openai import OpenAI
from anthropic import Anthropic

openai_client = OpenAI()
anthropic_client = Anthropic()
MODEL = "claude-3-haiku-20240307"
EMBED_MODEL = "text-embedding-3-small"


def embed(text: str) -> list[float]:
    return openai_client.embeddings.create(
        model=EMBED_MODEL, input=text
    ).data[0].embedding


def cosine_sim(v1: list[float], v2: list[float]) -> float:
    dot = sum(a * b for a, b in zip(v1, v2))
    m1 = math.sqrt(sum(a ** 2 for a in v1))
    m2 = math.sqrt(sum(b ** 2 for b in v2))
    return dot / (m1 * m2) if m1 and m2 else 0.0


# memory_store: {key: {"value": str, "embedding": list[float]}}
memory_store: dict = {}


def remember(key: str, value: str) -> None:
    """Store a memory with its embedding."""
    memory_store[key] = {"value": value, "embedding": embed(value)}


def recall(query: str, top_k: int = 3) -> list[tuple[str, str]]:
    """Return the top_k most relevant (key, value) pairs for the query."""
    if not memory_store:
        return []
    query_vec = embed(query)
    scored = [
        (key, data["value"], cosine_sim(query_vec, data["embedding"]))
        for key, data in memory_store.items()
    ]
    scored.sort(key=lambda x: x[2], reverse=True)
    return [(k, v) for k, v, _ in scored[:top_k]]


def answer_with_memory(question: str) -> str:
    relevant = recall(question, top_k=3)
    memory_context = "\n".join(f"- {k}: {v}" for k, v in relevant)
    system = f"You are a helpful assistant.\nRelevant memories:\n{memory_context}"
    response = anthropic_client.messages.create(
        model=MODEL,
        max_tokens=512,
        system=system,
        messages=[{"role": "user", "content": question}],
    )
    return response.content[0].text


# Usage
remember("user_name", "Alice")
remember("project", "Building a fraud detection pipeline in Python")
remember("bug_yesterday", "Fixed a race condition in the async task queue")
remember("preference", "Prefers concise answers, no Java examples")

print(answer_with_memory("What bug was I fixing?"))
# Retrieves the race condition memory and answers correctly
```

**When to use:** agents with many memories where injecting all of them would be too expensive or noisy. The retrieval step selects only what is relevant to the current query.

## Pattern 4: Memory Summarisation / Consolidation

When the conversation history grows too long, summarise the oldest messages into a compact block. This keeps the context window usable without losing important context.

```python
from anthropic import Anthropic

client = Anthropic()
MODEL = "claude-3-haiku-20240307"
MAX_MESSAGES = 20  # summarise when history exceeds this


def summarise_messages(messages: list[dict]) -> str:
    """Ask the LLM to produce a brief summary of a message list."""
    text = "\n".join(
        f"{m['role'].upper()}: {m['content']}" for m in messages
    )
    response = client.messages.create(
        model=MODEL,
        max_tokens=512,
        messages=[
            {
                "role": "user",
                "content": (
                    f"Summarise this conversation in 3-5 bullet points, "
                    f"preserving key facts and decisions:\n\n{text}"
                ),
            }
        ],
    )
    return response.content[0].text


def consolidate_if_needed(messages: list[dict]) -> list[dict]:
    """
    If messages exceeds MAX_MESSAGES, summarise the oldest half
    and replace them with a single summary message.
    """
    if len(messages) <= MAX_MESSAGES:
        return messages

    half = len(messages) // 2
    old_messages = messages[:half]
    recent_messages = messages[half:]

    summary = summarise_messages(old_messages)

    # Replace old messages with a system-style summary
    return [
        {
            "role": "user",
            "content": f"[Conversation summary]\n{summary}",
        },
        {
            "role": "assistant",
            "content": "Understood, I have the context from our earlier conversation.",
        },
    ] + recent_messages


# Usage
conversation = []
for i in range(25):
    conversation.append({"role": "user", "content": f"Message {i}"})
    conversation.append({"role": "assistant", "content": f"Reply {i}"})

conversation = consolidate_if_needed(conversation)
print(f"After consolidation: {len(conversation)} messages")
# Reduced from 50 to ~27 (summary block + recent half)
```

**When to use:** any agent with open-ended conversation length. Apply consolidation proactively before hitting the context limit, not reactively after.

## Anti-Patterns

<div className="antipattern">

**Storing everything forever — memory bloat**

```python
# Bad — every token from every conversation, indefinitely
for turn in conversation:
    memory_store.append(turn)

# Good — apply TTL or relevance scoring; discard low-value memories
# Keep the last N sessions, or memories above a confidence threshold
```

**Saving memories but never retrieving them**

```python
# Bad — memories are written but the system prompt never reads them
save_memory("user_name", "Alice")

# ... later ...
response = client.messages.create(
    model=MODEL,
    system="You are a helpful assistant.",  # memories never injected
    messages=messages,
)
```

**Storing sensitive PII in the memory store**

```python
# Bad — storing passwords, SSNs, credit card numbers as memories
remember("payment_card", "4111 1111 1111 1111")

# Good — never store sensitive credentials in agent memory;
# use a proper secrets manager. Memory stores are often logged.
```

</div>
