# Patterns: Agent Handoff

## Pattern 1: LLM-Based Intent Router

Classify a user message into one of N named intents using an LLM. Return the intent label so the caller can dispatch to the right handler.

```python
from anthropic import Anthropic

client = Anthropic()
MODEL = "claude-3-haiku-20240307"

INTENTS = {
    "code": "Questions about programming, debugging, code review, or software architecture",
    "database": "Questions about SQL, query optimisation, schema design, or database performance",
    "legal": "Questions about contracts, regulations, compliance, or legal interpretation",
    "general": "Any question that does not fit the above categories",
}


def classify_intent(message: str, intents: dict[str, str]) -> str:
    """
    Classify the user message into one of the provided intent labels.
    Returns the label string (e.g. 'code', 'database').
    """
    intent_list = "\n".join(f"- {label}: {description}" for label, description in intents.items())
    prompt = (
        f"Classify the following user message into exactly one of these categories:\n"
        f"{intent_list}\n\n"
        f"User message: {message}\n\n"
        "Respond with only the category label. No explanation."
    )
    response = client.messages.create(
        model=MODEL,
        max_tokens=32,
        messages=[{"role": "user", "content": prompt}],
    )
    label = response.content[0].text.strip().lower()
    # Validate — fall back to "general" if the label is not recognised
    return label if label in intents else "general"


# Usage
message = "My PostgreSQL query is taking 30 seconds. How do I add an index?"
intent = classify_intent(message, INTENTS)
print(intent)  # "database"
```

**When to use:** any system with 3 or more specialist agents where the routing logic cannot be captured by simple keyword matching. The LLM handles paraphrasing, ambiguity, and edge cases that rule-based routing misses.

## Pattern 2: Handoff with Context — Passing Conversation History

When the router delegates to a specialist, pass the full conversation history so the specialist has context for the current message.

```python
from anthropic import Anthropic

client = Anthropic()
MODEL = "claude-3-haiku-20240307"

SPECIALIST_SYSTEM_PROMPTS = {
    "code": (
        "You are an expert software engineer. "
        "Help with programming questions, debugging, code review, and architecture decisions."
    ),
    "database": (
        "You are a database expert specialising in SQL, query optimisation, and schema design. "
        "Provide precise, performance-focused answers."
    ),
    "general": "You are a helpful general-purpose assistant.",
}


def run_specialist(
    intent: str,
    message: str,
    conversation_history: list[dict],
) -> str:
    """
    Run the specialist agent for the given intent.
    Passes full conversation history for context.
    """
    system_prompt = SPECIALIST_SYSTEM_PROMPTS.get(intent, SPECIALIST_SYSTEM_PROMPTS["general"])

    # Build messages: prior history + new user message
    messages = list(conversation_history) + [{"role": "user", "content": message}]

    response = client.messages.create(
        model=MODEL,
        max_tokens=1024,
        system=system_prompt,
        messages=messages,
    )
    return response.content[0].text


def route_and_execute(
    message: str,
    intents: dict[str, str],
    conversation_history: list[dict] = None,
) -> str:
    """Classify intent and run the appropriate specialist."""
    if conversation_history is None:
        conversation_history = []

    intent = classify_intent(message, intents)
    return run_specialist(intent, message, conversation_history)


# Usage
history = [
    {"role": "user", "content": "I have a users table with 10 million rows."},
    {"role": "assistant", "content": "That is a significant dataset. What are you trying to query?"},
]
answer = route_and_execute(
    "The query is slow even with an index on email. Why?",
    INTENTS,
    conversation_history=history,
)
print(answer)
```

**When to use:** any multi-turn conversation where context from earlier turns is relevant to the specialist's answer. Passing history is almost always the right default.

## Pattern 3: Fallback Chain

Try the primary specialist, then a secondary one, then a general fallback. The user always gets an answer.

```python
from anthropic import Anthropic

client = Anthropic()
MODEL = "claude-3-haiku-20240307"


def specialist_response(system_prompt: str, message: str) -> str:
    response = client.messages.create(
        model=MODEL,
        max_tokens=1024,
        system=system_prompt,
        messages=[{"role": "user", "content": message}],
    )
    return response.content[0].text


def route_with_fallback(message: str, intent: str) -> str:
    """
    Try the primary specialist. If it declines or fails, try secondary, then general fallback.
    """
    specialist_chain = []

    if intent == "medical":
        specialist_chain = [
            ("Medical specialist", "You are a medical information assistant. Provide general health information only. If the question requires diagnosis or treatment, say: ESCALATE_TO_HUMAN"),
            ("General health", "You are a helpful assistant. For medical questions, provide general wellness information and recommend consulting a doctor."),
            ("General fallback", "You are a helpful assistant."),
        ]
    else:
        specialist_chain = [
            ("Primary", f"You are a specialist in {intent}."),
            ("General fallback", "You are a helpful assistant."),
        ]

    for name, system_prompt in specialist_chain:
        response = specialist_response(system_prompt, message)
        if "ESCALATE_TO_HUMAN" not in response:
            print(f"Handled by: {name}")
            return response

    return "I was not able to answer this question. Please contact a human specialist."


# Usage
answer = route_with_fallback("What is a safe dosage of ibuprofen?", intent="medical")
print(answer)
```

**When to use:** any system where some intents have restricted specialists (medical, legal) and you need an auditable trail of which handler responded.

## Anti-Patterns

<div className="antipattern">

**Routing without a fallback**

```python
# Bad — raises KeyError if intent is unrecognised
handler = specialist_registry[intent]

# Good — always have a default
handler = specialist_registry.get(intent, specialist_registry["general"])
```

**Losing context on handoff**

```python
# Bad — specialist only sees the immediate message
specialist_response(system=prompt, message=latest_message)

# Good — specialist sees the full conversation
specialist_response(system=prompt, messages=history + [{"role": "user", "content": latest_message}])
```

**Infinite handoff loops**

```python
# Bad — no depth limit; Specialist A -> B -> A -> B...
def handle(message, agent):
    if agent.should_delegate(message):
        return handle(message, agent.delegate_to)  # infinite recursion possible

# Good — enforce a handoff depth limit
def handle(message, agent, depth=0):
    if depth >= 1:
        return fallback_agent.respond(message)
    if agent.should_delegate(message):
        return handle(message, agent.delegate_to, depth=depth + 1)
    return agent.respond(message)
```
