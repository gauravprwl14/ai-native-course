# Chapter 27 — Agent Handoff

## What You'll Learn

- How a router agent classifies intent and selects the right specialist
- Handoff protocols: how to pass conversation context and a task summary to a specialist
- Graceful fallback: what to do when no specialist matches
- Confidence thresholds: when to escalate to a human instead of routing to an agent
- How to build an `AgentRouter` class that orchestrates intent classification and execution

## Prerequisites

- Chapter 24 (Multi-Agent) — you should understand how multiple agents can work together before building a routing layer on top
- Python 3.10+, `anthropic` package installed

## How This Connects

Chapter 24 (Multi-Agent) showed how agents collaborate. Agent Handoff is about *routing* — deciding which agent should handle a given request before any collaboration begins. Planning (Chapter 23) decomposes tasks into steps; routing selects the right executor for each step. Guardrails (Chapter 28) wrap the handoff to ensure no specialist receives unsafe or out-of-scope requests.
