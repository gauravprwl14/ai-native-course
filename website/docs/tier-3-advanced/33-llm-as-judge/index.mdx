## How This Connects

Chapter 32 gave you exact match and ROUGE-L — metrics for tasks where the correct answer is a specific string. But many LLM tasks don't have a single correct answer. "Is this customer support response helpful?" can't be answered with ROUGE. "Which of these two responses is better?" requires judgment.

LLM-as-Judge is the industry's answer: use a capable LLM with a carefully designed rubric to evaluate these subjective dimensions at scale. Used by OpenAI (in MT-Bench), Anthropic, and virtually every team that evaluates chat-model outputs in production.
