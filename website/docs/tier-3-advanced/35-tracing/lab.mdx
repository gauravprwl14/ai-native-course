# Lab: Build a Tracer

## Goal

Implement a lightweight tracing system in pure Python. No API calls needed — this lab is about data structures and timing logic.

## What You'll Build

| Class / Function | Description |
|-----------------|-------------|
| `Span` | Dataclass: name, model, input_tokens, output_tokens, latency_ms, error |
| `Trace` | Dataclass: list of spans + `total_tokens` and `total_latency_ms` properties |
| `calculate_cost_from_trace(trace)` | Sums input + output token costs for each span using `COST_PER_1K_TOKENS` |
| `create_span(name, fn, *args, **kwargs)` | Executes `fn(*args, **kwargs)`, records latency and errors, returns `(span, result)` |

## Acceptance Criteria

- `calculate_cost_from_trace` returns a float for a trace with known token counts
- `calculate_cost_from_trace` returns 0.0 for a trace with no spans or unknown models
- `create_span` returns a tuple of `(Span, result)`
- `create_span` records `latency_ms > 0` for any function call
- `create_span` captures exceptions in `span.error` without re-raising
- `Trace.total_tokens` sums input and output tokens across all spans
- `Trace.total_latency_ms` sums latency across all spans

## Hints

1. `calculate_cost_from_trace`: use `COST_PER_1K_TOKENS.get(span.model)` — if it returns `None`, skip that span.
2. `create_span`: use `time.time()` before and after the call. Calculate `(end - start) * 1000` for milliseconds.
3. `create_span` should catch all exceptions (`except Exception as e`), store `str(e)` in `span.error`, and return `(span, None)` rather than re-raising.
4. The `Trace` dataclass properties are already provided in the starter — no need to rewrite them.
