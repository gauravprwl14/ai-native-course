        "D) Use a single agent for all three tasks to avoid coordination overhead"
      ],
      correct: 1,
      explanation: "Steps 1 and 2 are independent — they can run in parallel using ThreadPoolExecutor, halving the latency. Step 3 depends on both and must wait for them to finish. Running all three in parallel would make step 3 start before its dependencies are ready. This is a classic fork-join pattern."

      explanation: "Python dicts are not thread-safe for concurrent writes. If two agents write to the same key at the same time, the result is a race condition — one write overwrites the other. The fix is to give each agent its own key in the shared state (state['research'], state['article']) or use thread-safe structures like queue.Queue."
