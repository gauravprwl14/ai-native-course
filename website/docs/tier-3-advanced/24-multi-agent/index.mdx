## How This Connects

Chapter 23 showed you how to decompose a goal into tasks. Multi-agent systems take this one step further: instead of one agent executing all tasks, you route each task to a *specialized agent* built exactly for that job. A researcher agent knows how to gather and organize information. A writer agent knows how to produce polished prose. An orchestrator coordinates between them.

The result is a system that scales: you can add new specialist agents without changing the orchestrator's core logic, and independent specialists can run in parallel, reducing end-to-end latency.
