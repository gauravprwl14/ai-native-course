# Lab 31 â€” Construct a DPO Dataset

In this lab you will implement a minimal DPO dataset construction pipeline that:

1. Creates and validates `(prompt, chosen, rejected)` preference pairs
2. Uses an LLM to generate synthetic rejected responses
3. Assembles a complete dataset from a list of prompts

## What You'll Build

| Function | Description |
|----------|-------------|
| `create_preference_pair(prompt, chosen, rejected)` | Returns a validated `{"prompt", "chosen", "rejected"}` dict |
| `validate_preference_pair(pair)` | Returns `(valid: bool, errors: list[str])` â€” checks keys, non-empty, chosen != rejected |
| `generate_rejection(prompt, chosen)` | Asks the LLM to generate a deliberately vague/unhelpful version of the chosen response |
| `build_dpo_dataset(prompts)` | For each prompt, generates a good response (chosen) and a bad response (rejected), returns a list of validated pairs |

## Your Tasks

### Task A: `create_preference_pair`

- Accept `prompt`, `chosen`, `rejected` as strings
- Return `{"prompt": prompt, "chosen": chosen, "rejected": rejected}`
- This is a pure constructor â€” keep it simple

### Task B: `validate_preference_pair`

- Check all three keys (`prompt`, `chosen`, `rejected`) exist in the dict
- Check none are empty strings (after strip)
- Check `chosen != rejected` (after strip)
- Return `(True, [])` if valid, or `(False, [list of error strings])` if not

### Task C: `generate_rejection`

- Build a prompt asking the model to generate a **worse** version of `chosen`
- Ask it to be vague, incomplete, or subtly inaccurate â€” not obviously nonsensical
- Call the Anthropic API with `temperature=0.7` to introduce some variety
- Return the model's response text

### Task D: `build_dpo_dataset`

- For each prompt in `prompts`:
  1. Call the LLM to generate a good (chosen) response
  2. Call `generate_rejection(prompt, chosen)` to get the rejected response
  3. Call `create_preference_pair(prompt, chosen, rejected)` to assemble the pair
  4. Validate the pair; skip any that fail validation
- Return the list of valid pairs
