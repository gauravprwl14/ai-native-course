# Lab: PII Detection and Redaction

## Goal

Build a PII handling module that detects, redacts, and pseudonymizes PII before it reaches an LLM API. All functions are pure string operations — no API calls needed.

## What You'll Build

| Function | Description |
|----------|-------------|
| `detect_pii(text)` | Run all `PII_PATTERNS` against text. Returns `{pii_type: [matched_values]}` |
| `redact_pii(text)` | Replace all PII with `[TYPE_REDACTED]` labels. Returns redacted string |
| `pseudonymize(text)` | Replace PII with consistent placeholders like `email_1`. Returns `(text, mapping)` |
| `restore_pseudonyms(text, mapping)` | Reverse pseudonymization using the mapping dict. Returns original text |

## Acceptance Criteria

- `detect_pii("Email me at john@example.com")` returns `{"email": ["john@example.com"]}`
- `detect_pii("SSN: 123-45-6789")` returns `{"ssn": ["123-45-6789"]}`
- `redact_pii("john@example.com")` returns a string containing `"[EMAIL_REDACTED]"`
- `redact_pii` does not contain the original email after redaction

## Hints

1. `detect_pii`: `re.findall` can return tuples if the pattern has capture groups. Clean this: `m if isinstance(m, str) else m[0]`.
2. `redact_pii`: iterate `PII_PATTERNS.items()` and apply `re.sub` sequentially.
3. `pseudonymize`: maintain two dicts: `original → placeholder` (for deduplication) and `placeholder → original` (the mapping to return).
4. `restore_pseudonyms`: a simple loop: `result = result.replace(placeholder, original)` for each entry.
