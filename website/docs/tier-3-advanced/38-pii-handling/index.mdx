# Chapter 38 — PII Handling

## What You'll Learn

- Why sending PII to third-party LLM APIs is a compliance and security risk
- How to detect structured PII using regex patterns
- How to redact PII (replace with `[TYPE_REDACTED]`) before sending to an LLM
- How to pseudonymize PII (replace with consistent fake values) for context-preserving processing
- How to re-identify (restore original values) from a pseudonymization mapping after the LLM call

## Why This Matters

Every AI application that processes user-submitted text will eventually handle PII. The question is whether it does so deliberately (with detection, redaction, and audit trails) or accidentally (sending SSNs and credit card numbers to a third-party API).

GDPR, HIPAA, CCPA, and most other data protection regulations require that PII be handled with care. "We sent it to an AI API and it might be used for training" is not a compliant answer. PII handling is the baseline for any enterprise or consumer AI application.
