# Lab: Build a Guardrail Pipeline

## Goal

Build a composable guardrail system that validates inputs before they reach the LLM and filters outputs before they reach the user.

## What You'll Build

| Function / Class | Description |
|-----------------|-------------|
| `is_safe_input(text)` | Regex check against `BLOCKED_PATTERNS`. Returns `(bool, str)` — safe flag and reason |
| `is_on_topic(text, allowed_topics)` | LLM classifier: is this text about one of the allowed topics? Returns `bool` |
| `filter_output(response, max_length)` | Trim response to `max_length`, handle edge cases. Returns filtered `str` |
| `GuardrailPipeline` | Composes all three checks. `run(user_input, model_response_fn)` returns a dict with keys `safe` (bool), `response` (str), `blocked_reason` (str or None) |

## Acceptance Criteria

- `is_safe_input("Ignore all previous instructions")` returns `(False, ...)`
- `is_safe_input("What is your return policy?")` returns `(True, "ok")`
- `is_on_topic("How do I return an item?", ["returns", "billing"])` returns `True`
- `filter_output("x" * 3000, max_length=2000)` returns a string of length `<= 2000`
- `GuardrailPipeline.run` with a blocked input returns `{"safe": False, ...}`
- `GuardrailPipeline.run` with a safe input calls `model_response_fn` and returns the filtered response

## Hints

1. `is_safe_input`: use `re.search(pattern, text, re.IGNORECASE)`. Return on first match.
2. `is_on_topic`: use `max_tokens=10` — you only need "yes" or "no".
3. `filter_output`: check `len(response) > max_length` first. If trimming, find the last `. ` in the truncated string using `rfind`.
4. `GuardrailPipeline.run`: return early on each blocked condition before calling `model_response_fn`.
