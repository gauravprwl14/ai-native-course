Lab: Model Benchmark Harness

Goal​

Build a benchmark harness that runs a set of test cases against multiple models (via a mock LLM caller), evaluates responses, and produces a comparison summary.

What You'll Build​

Data Classes​

ClassFields
TestCaseprompt: str, expected_keywords: list[str]
BenchmarkResultmodel, test_case_id, response, passed, latency_ms, input_tokens, output_tokens

Functions to Implement​

FunctionDescription
evaluate_response(response, expected_keywords)Returns True if ALL keywords appear in response.lower()
run_benchmark(test_cases, models, llm_caller)Runs each model against each test case via llm_caller(model, prompt) -> (response, latency_ms, in_tokens, out_tokens)
summarize_benchmark(results)Groups by model: returns dict with accuracy, avg_latency_ms, total_cost_estimate

Expected Output (from summarize_benchmark)​

{
    "claude-3-haiku-20240307": {
        "accuracy": 0.8,        # 80% of test cases passed
        "avg_latency_ms": 312.5,
        "total_cost_estimate": 0.000045
    },
    "claude-3-5-sonnet-20241022": {
        "accuracy": 1.0,        # 100% passed
        "avg_latency_ms": 850.0,
        "total_cost_estimate": 0.000540
    }
}

Acceptance Criteria​

  • evaluate_response("Paris is the capital of France", ["paris", "capital"]) returns True
  • evaluate_response("London is the capital", ["paris"]) returns False
  • run_benchmark(test_cases, models, mock_caller) returns one BenchmarkResult per model × test_case combination
  • summarize_benchmark(results) returns a dict with one entry per model containing accuracy, avg_latency_ms, total_cost_estimate
  • All tests pass with a mocked llm_caller (no real API calls)
  • pytest tests/ -v

How to Run​

cd curriculum/tier-4-architect/40-model-selection/lab
pytest tests/ -v

# Run manually
cd starter
python solution.py

Hints​

  1. evaluate_response should be case-insensitive: check kw.lower() in response.lower()
  2. run_benchmark receives a llm_caller function — call it as llm_caller(model, prompt) — this is what tests will mock
  3. For total_cost_estimate in summarize_benchmark, use a simple approximation: total_input_tokens / 1_000_000 * 1.0 + total_output_tokens / 1_000_000 * 3.0 (generic mid-tier pricing)
  4. accuracy in the summary should be a float between 0.0 and 1.0 (proportion of passed tests, not percentage)
