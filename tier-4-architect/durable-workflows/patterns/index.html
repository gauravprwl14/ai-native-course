Patterns: Durable Workflow Patterns

Pattern 1: File-Based Checkpointing​

The simplest durability pattern. After each step completes, serialize the entire workflow state to a JSON file. On startup, load the file and skip completed steps.

import json
from pathlib import Path

CHECKPOINT_DIR = Path("/tmp/workflows")

def save_checkpoint(workflow_id: str, state: dict) -> None:
    CHECKPOINT_DIR.mkdir(parents=True, exist_ok=True)
    path = CHECKPOINT_DIR / f"{workflow_id}.json"
    path.write_text(json.dumps(state, indent=2))

def load_checkpoint(workflow_id: str) -> dict | None:
    path = CHECKPOINT_DIR / f"{workflow_id}.json"
    if not path.exists():
        return None
    return json.loads(path.read_text())

def run_workflow(workflow_id: str, steps: list) -> list:
    state = load_checkpoint(workflow_id) or {
        "completed": [],
        "outputs": {}
    }

    results = []
    for name, func, args in steps:
        if name in state["completed"]:
            print(f"[SKIP] {name} (already done)")
            results.append(state["outputs"][name])
            continue

        print(f"[RUN]  {name}")
        output = func(*args)
        state["completed"].append(name)
        state["outputs"][name] = output
        save_checkpoint(workflow_id, state)
        results.append(output)

    return results

When to use: Small to medium workflows (under 50 steps) where a simple file store is acceptable. Great for scripts and local automation.

Limitations: Not suitable for concurrent workflows or distributed systems. Use a database-backed store for production.

Pattern 2: Idempotency Keys​

An idempotency key is a stable hash of (workflow_id, step_id). Before executing a step, check if a result exists for that key. This prevents re-execution even if the process crashes between completing the step and saving the checkpoint.

import hashlib

class IdempotencyStore:
    """In-memory store (replace with Redis or a database in production)."""

    def __init__(self):
        self._store: dict[str, any] = {}

    def make_key(self, workflow_id: str, step_name: str) -> str:
        raw = f"{workflow_id}:{step_name}"
        return hashlib.sha256(raw.encode()).hexdigest()[:16]

    def get(self, workflow_id: str, step_name: str):
        key = self.make_key(workflow_id, step_name)
        return self._store.get(key)

    def set(self, workflow_id: str, step_name: str, value) -> None:
        key = self.make_key(workflow_id, step_name)
        self._store[key] = value

    def exists(self, workflow_id: str, step_name: str) -> bool:
        key = self.make_key(workflow_id, step_name)
        return key in self._store


store = IdempotencyStore()

def run_step_idempotent(workflow_id: str, step_name: str, func, *args):
    if store.exists(workflow_id, step_name):
        print(f"[CACHED] {step_name}")
        return store.get(workflow_id, step_name)

    result = func(*args)
    store.set(workflow_id, step_name, result)
    print(f"[DONE]   {step_name}")
    return result

When to use: Any step that calls external APIs, charges money, sends emails, or writes to a database. Idempotency is non-negotiable for these operations.

Pattern 3: Step-Level Retry with Max Retries​

Retry individual failed steps with exponential backoff, not the entire workflow. The workflow continues from where the failure occurred.

import time

def run_with_retry(func, *args, max_retries: int = 3, backoff_base: float = 1.0):
    """Run func(*args) with exponential backoff retry."""
    last_error = None
    for attempt in range(max_retries + 1):
        try:
            return func(*args)
        except Exception as e:
            last_error = e
            if attempt < max_retries:
                wait = backoff_base * (2 ** attempt)
                print(f"[RETRY] attempt {attempt + 1}/{max_retries}, waiting {wait}s: {e}")
                time.sleep(wait)
            else:
                print(f"[FAILED] max retries exceeded: {e}")
    raise last_error


def run_workflow_with_retry(workflow_id: str, steps: list) -> list:
    state = load_checkpoint(workflow_id) or {"completed": [], "outputs": {}}
    results = []

    for name, func, args, max_retries in steps:
        if name in state["completed"]:
            results.append(state["outputs"][name])
            continue

        try:
            output = run_with_retry(func, *args, max_retries=max_retries)
            state["completed"].append(name)
            state["outputs"][name] = output
            save_checkpoint(workflow_id, state)
            results.append(output)
        except Exception as e:
            print(f"[ABORT] workflow stopped at step '{name}': {e}")
            state["status"] = "failed"
            save_checkpoint(workflow_id, state)
            break

    return results

When to use: Any step that calls external services (LLM APIs, web requests, databases) where transient failures are expected. Rate limit errors and network timeouts are the most common triggers.

Pattern 4: Saga Pattern with Compensating Transactions​

Each step registers a compensation (undo) function. If the workflow fails, completed steps run their compensations in reverse order to restore a consistent state.

from dataclasses import dataclass, field
from typing import Callable, Any

@dataclass
class SagaStep:
    name: str
    action: Callable
    compensation: Callable
    args: list = field(default_factory=list)

class SagaWorkflow:
    def __init__(self):
        self.steps: list[SagaStep] = []
        self.completed: list[SagaStep] = []

    def add_step(self, step: SagaStep) -> "SagaWorkflow":
        self.steps.append(step)
        return self

    def run(self) -> list[Any]:
        results = []
        for step in self.steps:
            try:
                print(f"[ACTION] {step.name}")
                result = step.action(*step.args)
                self.completed.append(step)
                results.append(result)
            except Exception as e:
                print(f"[FAIL] {step.name}: {e}")
                self._compensate()
                raise
        return results

    def _compensate(self):
        print("[COMPENSATING] rolling back completed steps...")
        for step in reversed(self.completed):
            try:
                print(f"[UNDO] {step.name}")
                step.compensation()
            except Exception as e:
                print(f"[WARN] compensation for {step.name} failed: {e}")


# Example usage
saga = SagaWorkflow()
saga.add_step(SagaStep(
    name="reserve_data",
    action=lambda: print("data reserved"),
    compensation=lambda: print("data reservation cancelled")
))
saga.add_step(SagaStep(
    name="generate_report",
    action=lambda: print("report generated"),
    compensation=lambda: print("report deleted")
))
saga.add_step(SagaStep(
    name="publish_report",
    action=lambda: (_ for _ in ()).throw(RuntimeError("publish failed")),
    compensation=lambda: print("report unpublished")
))

try:
    saga.run()
except RuntimeError:
    pass  # compensations already ran

When to use: Workflows that modify external systems (databases, storage, APIs) where partial completion leaves the system in an inconsistent state.

Pattern 5: LangGraph State Graph​

LangGraph models workflows as a directed graph where each node is a step function. State is automatically persisted between nodes using a checkpointer.

from typing import TypedDict
from langgraph.graph import StateGraph, END
from langgraph.checkpoint.memory import MemorySaver

class WorkflowState(TypedDict):
    goal: str
    raw_data: list[str]
    summary: str
    report: str

def fetch_data(state: WorkflowState) -> WorkflowState:
    """Step 1: fetch raw data."""
    data = [f"source_{i}" for i in range(5)]  # replace with real fetches
    return {"raw_data": data}

def summarise(state: WorkflowState) -> WorkflowState:
    """Step 2: summarise data with LLM."""
    summary = f"Summary of {len(state['raw_data'])} sources"
    return {"summary": summary}

def write_report(state: WorkflowState) -> WorkflowState:
    """Step 3: write final report."""
    report = f"Report based on: {state['summary']}"
    return {"report": report}

# Build the graph
builder = StateGraph(WorkflowState)
builder.add_node("fetch_data", fetch_data)
builder.add_node("summarise", summarise)
builder.add_node("write_report", write_report)

builder.set_entry_point("fetch_data")
builder.add_edge("fetch_data", "summarise")
builder.add_edge("summarise", "write_report")
builder.add_edge("write_report", END)

# MemorySaver persists state in memory; use SqliteSaver for disk persistence
checkpointer = MemorySaver()
graph = builder.compile(checkpointer=checkpointer)

# Each run is identified by thread_id — allows resume
config = {"configurable": {"thread_id": "workflow-run-001"}}
result = graph.invoke({"goal": "research AI trends"}, config=config)
print(result["report"])

When to use: LLM-heavy workflows with conditional branching, cycles (retry loops), or where you want a visual graph representation of your pipeline.

Anti-Patterns​

Anti-patternWhat Goes WrongFix
No checkpointingAny crash restarts the entire workflow; wasted time and API costCheckpoint after every step; even a simple JSON file dramatically improves reliability
Checkpointing too frequentlySaving state after every LLM token or every loop iteration adds significant overheadCheckpoint at step boundaries only, not within a step
Storing entire LLM responsesCheckpoints balloon to hundreds of MB; slow to read/writeStore summaries or output IDs; re-fetch full content if needed
No idempotency on external callsA resume re-sends emails, re-charges cards, re-writes database rowsIdempotency key on every step that mutates external state
Retrying the entire workflowSteps 1 and 2 succeed; step 3 fails; steps 1 and 2 run again on retryStep-level retry only; skip completed steps on resume
